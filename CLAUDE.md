## Project Overview

Newt is a language-conditioned multitask world model for continuous control based on TD-MPC2. The repo also contains **MMBench**, a suite of 200 continuous control tasks across 9 domains. This is a research codebase—no formal test suite exists.

## Key Commands

```bash
# Environment setup (already setup)
conda activate newt

# Training (runs from repo root, working dir is tdmpc2/)
python train.py                          # 20M model, all 200 tasks ("soup")
python train.py model_size=B task=walker-walk  # single-task (5M params)
python train.py model_size=XL            # 80M model
python train.py obs=rgb                  # state+RGB observations
python train.py checkpoint=path/to/ckpt.pt  # resume from checkpoint
python train.py multiproc=True           # multi-GPU (DDP, uses all visible GPUs)

# Download pretrained checkpoints from HuggingFace
python download_checkpoints.py --filename "walker-walk" --cache-dir="./checkpoints"
python download_checkpoints.py --all --cache-dir="./checkpoints"

# Generate demonstrations
python generate_demos.py task=walker-walk +num_demos=10 data_dir=/path/to/data
```

Configuration uses Hydra—parameters are passed as `key=value` CLI args. See `tdmpc2/config.py` for all options.

## Architecture

### Training Pipeline

1. **Config** (`tdmpc2/config.py`): `Config` dataclass parsed by `parse_cfg()`. Loads task metadata (embeddings, action dims, episode lengths) from `tasks.json`. Model sizes: S(2M), B(5M), L(20M), XL(80M).

2. **Entry point** (`tdmpc2/train.py`): Sets up buffer, optionally loads demos, spawns DDP workers for multi-GPU. The `train()` function creates the WorldModel, wraps it in TDMPC2 agent, and hands off to Trainer.

3. **WorldModel** (`tdmpc2/common/world_model.py`): Encoder (LayerNorm+ReLU MLP) → latent dynamics (SimNorm), reward head (discrete regression), policy prior, Q-function ensemble. Task conditioning via frozen CLIP text embeddings (512-dim). Action masking per task.

4. **TDMPC2 Agent** (`tdmpc2/tdmpc2.py`): Wraps WorldModel with MPPI planning (`_mppi()`), policy prior (`_pi()`), and training losses (`_loss_fn()` for world model, `_pi_loss()` for policy). Separate optimizers for model and policy.

5. **Trainer** (`tdmpc2/trainer.py`): Main loop—demo pretraining → online training with environment rollouts, buffer management, evaluation. Tracks rewards, episode lengths, success rates, normalized scores.

6. **Replay Buffer** (`tdmpc2/common/buffer.py`): torchrl-based. `EnsembleBuffer` combines offline (demo) and online buffers. Trajectory sampling with configurable horizon.

### Key Concepts

- **"soup"**: The multitask setting—all 200 tasks trained jointly. `task=soup` is the default. Single-task disables demo loading and task conditioning.
- **Task embeddings**: Frozen CLIP text embeddings stored in `tasks.json`. Generated by `tdmpc2/create_task_embeddings.py`.
- **Discrete regression**: Q-values and rewards predicted as distributions over bins (101 bins by default), not scalar values.
- **MPPI planning**: Model Predictive Path Integral—samples 512 action sequences, refines with 64 elites over 6 iterations at planning horizon 3.
- **DDP**: Multi-GPU splits tasks across GPUs by rank (round-robin). Number of tasks must be divisible by GPU count.

## Experiment Conventions

Experiment setup, folder structure, SLURM submission, eval logging, and git branching are defined in `.claude/skills/experiment/SKILL.md`. Refer to that skill for the full workflow.

### CSV Format
- All metric CSVs use: snake_case columns, first column is always `step`, second is always `task`.

### Naming
- Scripts: descriptive verb, e.g. `evaluate_esnr.py`, `compute_e_sim.py`
- Plots: `<metric_name>_<task>.pdf`

### Wandb
Log training runs to: `pair-diffusion/world_model_metric_analysis/<run_id>`. Set `wandb.job_name` to the experiment name. Metric-only analysis does not require wandb — use local CSVs and plots.

### Environment System

`tdmpc2/envs/__init__.py` → `make_env()` creates vectorized environments. Each domain has its own module (`dmcontrol.py`, `metaworld.py`, `maniskill.py`, `atari.py`, `box2d.py`, `mujoco.py`, `pygame.py`, `robodesk.py`, `ogbench.py`). Custom tasks live in `tdmpc2/envs/tasks/`, wrappers in `tdmpc2/envs/wrappers/`.

Task prefixes map to domains: `mw-` = MetaWorld, `ms-` = ManiSkill, `og-` = OGBench, `rd-` = RoboDesk, `pygame-` = Pygame, `atari-` = Atari, `mujoco-` = MuJoCo, `bipedal-`/`lunarlander-` = Box2D. Unprefixed = DMControl.

## Dependencies

Python 3.10, PyTorch 2.8, CUDA required. Conda env defined in `docker/environment.yaml`. Key deps: torchrl, tensordict, dm-control, mani_skill, metaworld (custom fork), gymnasium, hydra-core, transformers (for CLIP), wandb.

ManiSkill assets must be downloaded separately to `~/.maniskill/`.
